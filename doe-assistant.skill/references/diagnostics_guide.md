# 진단 4종 — 동료가 읽을 수 있는 가이드

진단은 **차단형 게이트가 아니다.** 항상 실행하고, 결과의 *의미*를 일상어로 풀어주고, 결론의 어조를 조절하는 데 쓴다.

각 진단은 다음 5블록으로 보고한다 — 일상어 풀이가 본체로 먼저 오고, 통계 어휘는 마지막 라벨에 모아 보존한다(근거 추적·인용 용도).

1. **이걸 왜 보는가** — 동료 입장에서 한두 문장 일상어
2. **결과 보는 법** — 통과 / 약함 / 위반 각각이 일상어로 무엇을 의미
3. **공정에서 보면** — 이번 결과가 동료의 공정 결정에 어떤 의미
4. **결론 어조 영향** — 어떻게 보수화할지
5. *(통계적 근거 라벨)* — 어휘·수치를 함께 박아 근거 자료로 보존

---

## 1. 오차의 흩어짐이 자연스러운가

**이걸 왜 보는가**
모델이 예측한 값과 실제 측정값 사이의 *벗어남(오차)* 들을 모아보면, 보통은 *"0 근처에 많이 모이고 양 끝으로 갈수록 적어지는 종 모양"* 이 됩니다. 종 모양이 자연스러우면 모델이 잘 맞은 거고, 한쪽으로 비뚤어졌거나 꼬리가 길면 모델이 놓친 패턴이 있다는 신호.

**결과 보는 법**
- **통과** — 오차가 자연스럽게 흩어짐. 우연한 노이즈로 봐도 됨.
- **약함** — 한쪽으로 살짝 치우치거나 꼬리가 김. 한두 wafer에서 예상 못한 일이 있었던 흔적일 가능성.
- **위반** — 종 모양에서 많이 벗어남. 모델이 데이터의 큰 흐름을 놓치고 있을 수 있음.

**공정에서 보면**
한쪽으로 치우침이 보이면 그 방향의 outlier wafer를 한 번 따로 점검할 가치가 있습니다. 가스 라인 일시 이상, plasma 점화 불안정, 측정 장비 드리프트 같은 일시적 외란이 흔한 원인이에요.

**결론 어조 영향**
- 통과 → 단정 톤 가능
- 약함 → "효과 방향성은 신뢰. 절대 수치는 약간 흔들릴 수 있음"
- 위반 → "효과 순서까지만 신뢰. 절대 수치는 다음 실험으로 확인 필요"

*(통계적 근거 라벨: Shapiro-Wilk 검정. W 통계량과 p-value. 보통 p>0.05면 정규성 가정 통과로 본다. 잔차의 정규성은 ANOVA p-value의 정확도와 직결됨.)*

## 2. 모델 어긋남에 모양이 있는가

**이걸 왜 보는가**
모델이 *어디서 더 잘 맞고 어디서 어긋나는지* 를 보는 그림이에요. 모델이 영역 전체에 잘 맞으면 어긋남이 어디서나 비슷한 크기로 *무작위*로 흩어집니다. 어긋남에 *모양*이 보이면 — 모델이 못 잡은 무언가가 그 모양에 숨어 있다는 뜻.

**결과 보는 법 — 어긋남의 모양 세 가지**

| 보이는 모양 | 일상어 의미 |
|---|---|
| **활처럼 휜 모양** | 직선 모델로 끝났는데 실제 공정은 *휘어 있는 반응* 이라는 뜻. 곡선 반응이 있음 |
| **나팔 모양** (한쪽으로 갈수록 점점 흩어짐 폭이 커짐) | 어떤 영역(보통 큰 값쪽)에선 결과가 더 들쭉날쭉. wafer-to-wafer 편차가 큰 영역이 있음 |
| **줄무늬·계단 모양** | 우리가 *기록 안 한 변수* 가 결과에 영향 주고 있다는 신호. 시간 순서, wafer 위치, gas line 컨디션, operator 등이 흔한 후보 |

**공정에서 보면**
- 활처럼 휘었다면 → 다음 라운드에서 RSM(곡선까지 잡는 추가 실험)이 자연스러운 길
- 나팔이라면 → 들쭉날쭉한 영역(보통 극단 조건)은 양산 회피, 또는 그 영역에서 추가 robustness 점검
- 줄무늬라면 → 동료에게 *"혹시 측정 시간대, wafer 위치, gas 라인 컨디션, 마지막 wet clean 후 시간 같은 게 영향을 줄 수 있을까요?"* 물어보고 후보 변수를 다음 실험에 추가

**결론 어조 영향**
- 패턴 없음 → 단정 톤
- 활처럼 휨 → "이번 직선 가정에서 이 정도이고, 실제로는 곡선 반응 가능성. 다음 라운드 RSM 권장"
- 나팔 → "효과 크기는 살아있지만 큰 값 영역의 정확도는 낮음"
- 줄무늬 → "누락 변수 가능성. 다음 실험 전에 점검 필요"

*(통계적 근거 라벨: Residual vs Fitted plot 시각 점검 + 보조로 Breusch-Pagan(분산 일정성) 또는 White's test의 p-value.)*

## 3. 조건마다 흔들림이 비슷한가

**이걸 왜 보는가**
같은 조건을 여러 번 측정하면 wafer마다 결과가 살짝 다른 게 보통이에요(wafer-to-wafer 편차). 이 도구는 *조건마다 그 편차의 크기가 비슷한지* 봅니다. 어떤 조건은 매번 비슷한 결과인데 어떤 조건은 들쭉날쭉하다면 — 그 들쭉날쭉한 조건은 양산에 위험.

**결과 보는 법**
- **통과** — 모든 조건이 비슷한 안정성
- **위반** — 일부 조건이 다른 조건보다 더 흔들림

**공정에서 보면**
*어느 조건에서* 흔들림이 큰지가 핵심. 흔히 고온·고압·고RF 같은 극단 조건에서 plasma stability가 약해져 흔들림이 커지는 경향이 있어요. 양산용 조건은 흔들림이 작은 영역에서 우선 고르고, 흔들림 큰 조건을 어쩔 수 없이 써야 한다면 추가 robustness 점검(반복 측정)으로 양산 가능성 확인.

**결론 어조 영향**
- 통과 → 단정 톤
- 위반 → "효과 평균은 신뢰. 단, 흔들림 큰 조건(예: 고온·고압)은 양산 적용 전 반복 측정으로 robustness 확인 필요"

*(통계적 근거 라벨: Levene's test. F 통계량과 p-value. 보통 p>0.05면 등분산 가정 통과.)*

## 4. 직선만으론 부족한가 (center points 활용)

**이걸 왜 보는가**
보통 factorial 실험은 양 끝값만 봅니다 — 예를 들어 온도 150°C와 200°C만 측정. 그 사이(175°C)는 *"직선으로 이어진다고 가정"* 하고 추정해요. 그런데 *실험 중에 가운데값(175°C)에서도 따로 측정해 두었다면* (= center points), 그 가운데 측정값이 직선 추정과 같은지 다른지를 직접 비교할 수 있어요. 다르면 직선이 아니라 *곡선 반응*이 있다는 명확한 증거.

**결과 보는 법**
- **통과** — 가운데값이 직선 추정과 거의 같음. 이 영역에서 직선 모델로 충분.
- **위반** — 가운데값이 직선 추정에서 벗어남. **곡선 반응 있음.**

**공정에서 보면**
곡선 반응이 있다는 건 — *최적 조건이 양 끝이 아니라 가운데 어딘가에 있을 가능성*이 높다는 뜻이에요. 이건 DOE에서 가장 중요한 다음 실험 신호 중 하나입니다. 양 끝만 본 1차 라운드에서 *"바깥으로 더 가야 하나"* 헷갈리던 부분이, 사실 *"가운데 안쪽"* 이 답일 가능성을 가리키는 거죠. RSM(안전한 범위에서 곡선까지 잡는 추가 실험)으로 정밀화하는 게 자연스러운 다음 단계.

이건 ChatGPT가 가장 자주 빠뜨리는 진단이라서, 자가검증 메모에 거의 항상 등장합니다.

**결론 어조 영향**
- 통과 → "이 영역에선 직선 모델로 충분"
- 위반 → "곡선 반응이 잡혔습니다. 최적이 코너 바깥보다 *안쪽 어딘가*에 있을 가능성이 높음. 다음 라운드는 RSM(Box-Behnken 또는 CCD) 권장"

*(통계적 근거 라벨: Lack-of-fit F-test (center points 활용). F 통계량과 p-value. p<0.05면 곡률(quadratic curvature) 존재 가능성. RSM 권장 트리거.)*

## 종합 어조 매트릭스

진단 4종 중 통과한 개수에 따라 결론 어조를 조절한다.

| 통과 | 결론 어조 | 다음 실험 톤 |
|---|---|---|
| 4/4 | 단정 가능 | 자연스러운 다음 단계 권장 |
| 3/4 | 효과 방향 확실, 절대 수치 잠정 | 권장하되 약한 진단 항목 보강 묶어서 |
| 2/4 | 효과 순서까지만 신뢰 | 권장 + "결론 확정 전 추가 실험 필요" |
| 0-1/4 | 결론 잠정 | "이번 라운드는 탐색으로 보고 다음 라운드에서 정밀화" |

**어느 경우에도 결론 절을 *삭제*하지 않는다.** 어조만 바꾼다. 동료가 결정을 내릴 수 있도록 정보는 항상 제공한다.

## 진단별 다음-스텝 매핑 (위반 패턴 → design 분기)

통과 개수만으론 부족하다. **어느 진단이 위반됐는지**에 따라 다음 스텝 design 분기가 달라진다.

이 표는 *LLM이 빠뜨릴 만한 분기 후보*를 reminder하는 출발점이다. 강제 SOP가 아니라 — **이 표에 없는 패턴이 더 자연스러우면 표를 따르지 말 것**. 케이스의 도메인 맥락이 표보다 우선한다. 본문 §6 다음 스텝 절은 이 표를 *방향 제시*로만 쓰고, 동료가 어느 가설을 더 의심하는지 물어 분기를 좁힌다.

### 단독 위반

| 위반 진단 | 무엇이 의미되는가 | 결론 어조 | 다음 스텝 트리거 |
|---|---|---|---|
| Shapiro (잔차 정규성) | p-value 정확도 흔들림 | 효과 방향성 OK, 절대 수치 보수화 | outlier 점검 → 변환(log·sqrt) 시도 → replicate 늘려 재추정. **design 자체는 유지** |
| Residual vs fitted — 곡선 패턴 | 선형 모델 부족, 곡률 존재 | 곡률 영역 명시 | **RSM 직진** (Box-Behnken / CCD) |
| Residual vs fitted — 깔때기 | 분산이 예측값과 함께 커짐 | 큰 영역 정확도 약함 | **분산 안정화 변환**(log) 또는 가중 회귀. design 유지 |
| Residual vs fitted — 줄무늬·계단 | 누락 factor 신호 | "누락 변수 가능성" 명시 | **Plackett-Burman 스크리닝**으로 후보 factor 거름 (chamber position·time-since-clean·gas·operator 등) |
| Levene (등분산성) | 일부 조건 잡음 큼 | 양산 적용 전 robustness 점검 톤 | 동일 조건 **replicate 5회** / **Taguchi robust design** |
| Lack of fit (center points) | 모델 외 곡률 존재 | 최적점 위치 잠정 톤 | **RSM 직진** — 가장 명확한 트리거 |

### 조합 패턴 (실전에서 자주 나옴)

| 위반 조합 | 해석 | 권장 다음 스텝 |
|---|---|---|
| Lack-of-fit + Levene | 곡률 잡혔지만 일부 조건 잡음 큼 | **외란 거른 뒤 RSM**. Plackett-Burman 12 run 스크리닝 → Box-Behnken |
| Residual 패턴 + Lack-of-fit | 곡률이 진짜인지 *누락 factor* 효과인지 모호 | **Plackett-Burman 우선** — 누락 factor 거른 뒤 RSM 결정 |
| Shapiro + Residual 패턴 | 잔차 분포·구조 모두 흔들림 | 데이터 품질 점검 우선. outlier·변환 시도 후 재분석. **design 변경 보류** |
| Lack-of-fit + Residual 곡선 | 곡률 시그널이 두 진단에서 모두 잡힘 | **RSM 강하게 권장** — 망설이지 말 것 |
| Levene + Shapiro | 분산 + 정규성 동시 흔들림 | log 변환 시도 → 변환 후에도 위반이면 반복 측정 + Taguchi |

### 적용 규칙

- 단독 위반인지 조합인지 먼저 판별. 조합이 더 강한 신호.
- 본문 §6에선 **주 권장 1개 + 대안 시나리오 1-2개**를 함께 적는다.
- 동료가 *어느 가설을 더 의심하는지* 물어 분기 좁힘 — 단독 결정 강요 회피.
- 매핑 표에 없는 위반 패턴이 보이면 — 표를 박지 말고 *어시스턴트 톤*으로 풀어 설명한다. 이 표는 출발점이지 강제 SOP가 아니다.
